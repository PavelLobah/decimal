# decimal.h

### Арифметические операторы

| Название оператора | Оператор  | Функция                                                                            | 
| ------ | ------ |------------------------------------------------------------------------------------|
| Сложение | + | int s21_add(s21_decimal value_1, s21_decimal value_2, s21_decimal *result)         |
| Вычитание | - | int s21_sub(s21_decimal value_1, s21_decimal value_2, s21_decimal *result)         |
| Умножение | * | int s21_mul(s21_decimal value_1, s21_decimal value_2, s21_decimal *result) | 
| Деление | / | int s21_div(s21_decimal value_1, s21_decimal value_2, s21_decimal *result) |
| Остаток от деления | Mod | int s21_mod(s21_decimal value_1, s21_decimal value_2, s21_decimal *result) |

Функции возвращают код ошибки:  
- 0 - OK  
- 1 - число слишком велико или равно бесконечности
- 2 - число слишком мало или равно отрицательной бесконечности
- 3 - деление на 0

### Операторы сравнение

| Название оператора | Оператор  | Функция | 
| ------ | ------ | ------ |
| Меньше  | < | int s21_is_less(s21_decimal, s21_decimal) |
| Меньше или равно | <= | int s21_is_less_or_equal(s21_decimal, s21_decimal) | 
| Больше | \> |  int s21_is_greater(s21_decimal, s21_decimal) |
| Больше или равно | \>= | int s21_is_greater_or_equal(s21_decimal, s21_decimal) | 
| Равно | == |  int s21_is_equal(s21_decimal, s21_decimal) |
| Не равно | != |  int s21_is_not_equal(s21_decimal, s21_decimal) |

Возвращаемое значение:
- 0 - FALSE
- 1 - TRUE

## Реализация функций библиотеки decimal.h

- Для запуска программы сборки и тестирования используйте `make test` в терминале
- Библиотека, разработанная на языке C стандарта C11 с использованием компилятора gcc
- Не использовать устаревшие и устаревшие языковые конструкции и библиотечные функции. Используется стандарт POSIX.1-2017.
- Это статическая библиотека (с заголовочным файлом decimal.h)
- Библиотека, разработанная в соответствии с принципами структурированного программирования;
- Подготовлено полное покрытие кода библиотечных функций модульными тестами с использованием библиотеки Check
- Модульные тесты охватывали не менее `80%` каждой функции (проверено с помощью gcov) 
- Предоставлен Makefile для сборки библиотеки и тестов (с целями all, clean, test, decimal.a, gcov_report) 
- Цель gcov_report генерирует отчет gcov в виде HTML-страницы.
- Не используйте тип __int128
- Данный тип поддерживает числа от -79,228,162,514,264,337,593,543,950,335 до +79,228,162,514,264,337,593,543,950,335.
